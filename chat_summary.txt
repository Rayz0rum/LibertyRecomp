Summary
Adds a library of native CRT hook implementations for recompiled Xbox 360 projects, runtime XAM ordinal resolution, and improvements to the codegen and CLI tooling.

Statically-linked CRT functions (memcpy, CreateFileA, RtlAllocateHeap, etc.) are identified by address in the project TOML config. Codegen strips them from the recompilation pass and wires in SDK-provided native implementations automatically.

CRT Hooks
[rexcrt] config table parsed by codegen generates declarations and skips recompilation of hooked functions, linking them against the SDK rexkernel instead.
o1heap-backed deterministic allocator living in guest address space, auto-initialized when the heap function group is present in [rexcrt].
51 hookable functions across 4 categories:
File I/O (24): CreateFileA, ReadFile, WriteFile, CloseHandle, FindFirstFileA, FindNextFileA, FindClose, SetFilePointer, SetFilePointerEx, GetFileSize, GetFileSizeEx, GetFileAttributesA, GetFileAttributesExA, SetFileAttributesA, SetEndOfFile, FlushFileBuffers, DeleteFileA, CreateDirectoryA, RemoveDirectoryA, MoveFileA, SetFileTime, CompareFileTime, CopyFileA, GetFileType
Heap (4): RtlAllocateHeap, RtlFreeHeap, RtlSizeHeap, RtlReAllocateHeap
Memory (10): memcpy, memmove, memset, memchr, XMemCpy, XMemSet, XMemSet128, memset_vmx, memcpy_s, memmove_s
String (13): strncmp, strncpy, strchr, strstr, strrchr, strtok, _stricmp, strcpy_s, lstrlenA, lstrcpyA, lstrcpynA, lstrcatA, lstrcmpiA
Any hook can be replaced by defining the same rexcrt_<Name> symbol in the consumer project (standard static library link-order resolution, no weak symbols needed).
XAM Ordinal Resolution
Runtime ordinal-to-function resolution for XAM and xboxkrnl exports.
Global PPC function registry populated by static constructors from XAM_EXPORT / XBOXKRNL_EXPORT macros.
Thunk allocation system in the processor: assigns guest-addressable thunk addresses for host-side PPC functions so they can be returned from GetProcAddress at runtime.
Thunk cache per kernel module avoids duplicate allocations.
Codegen and CLI Improvements
PPC image runtime config (PPCImageInfo) is now a codegen output. A new {project}_config.cpp is generated containing code base, code size, image base, image size, function mappings, and rexcrt heap flag. Consumer projects no longer define these values manually.
App boilerplate relocated from main.cpp into template_utils.h so that rexglue migrate no longer overwrites user app code.
rexglue migrate now regenerates rexglue.cmake and the app header from templates, keeping user main.cpp and CMakeLists.txt intact.
Version tracking: CMake configure_file generates rex/version.h from project version. CLI and codegen output now embed REXGLUE_VERSION_STRING.
File I/O hooks are no longer enforced as an all-or-nothing group. Individual file hooks can be specified independently (heap group is still enforced).
Additional Changes
IDA Pro helper script (scripts/ida/export_named_funcs.py) for extracting user-named functions from an IDB and updating a ReXGlue TOML config.
Version bumped to 0.3.0.
What's included
Heap hooks

RtlAllocateHeap/Free/Size/ReAlloc backed by o1heap.
Supports HEAP_ZERO_MEMORY, realloc with zero-fill of new bytes, and safe free of out-of-heap pointers.
Exposes diagnostics (capacity, allocated, peak, OOM count).
File I/O hooks

Win32-style wrappers (CreateFileA, ReadFile, WriteFile, CloseHandle, FindFirstFileA/FindNextFileA/FindClose, etc.) backed by the existing VFS.
Handles OVERLAPPED reads/writes, seeking, file size queries, directory creation, file copy/move/delete, and directory enumeration.
XAM ordinal support

GetProcAddressByOrdinal now resolves exports at runtime by looking up the global PPC function registry and allocating guest thunks on demand.
Enables multi-module titles that call XAM functions by ordinal rather than by compile-time linking.
Codegen pipeline

Parses [rexcrt] table from project TOML, renames function graph nodes to rexcrt_<Name>.
Emits extern declarations, mapping table entries, and PPCImageConfig struct.
Removes rexcrt functions from the recompilation pass.
Generates {project}_config.cpp with image layout and rexcrt flags.
String and memory hooks

Direct native implementations of all 23 string/memory functions listed above.
Portable across Windows and Linux.
Usage
Populate your TOML file with the CRT function name and the function address from your project.

[rexcrt]
CreateFileA     = 0x8248B780
memcpy          = 0x82601000
# ... and others
Notes on usage
Codegen validates that certain rexcrt function sets are all-or-nothing.
If you specify any function in the heap group, you must specify all 4 (RtlAllocateHeap, RtlFreeHeap, RtlSizeHeap, RtlReAllocateHeap). Missing hooks would leave unresolved targets since the originals are stripped from recompilation.
Any hook can be project-overridden by defining the same rexcrt_<Name> symbol in your project.
